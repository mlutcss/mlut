3 #core-blocks Core blocks

Set of universal blocks for using on any project. Some blocks contain elements and modifiers.

Core blocks don't contain, or almost don't contain styles such as colors, backgrounds, etc. Such styles are set on the theme or 'design' layer. To style blocks or their elements, use utilities or modifiers such as block--mod.

Block for debugging layouts that built with grid system. It displays a grid system columns and gutters in the background of the container. The grid system are generated depending on the settings.

There may be minor inaccuracies in the drawing of the grid, because the grid is generated with CSS linear-gradient(). For gutters calculation are used CSS custom properties without fallback.

3.4 #core-blocks.row Row

Toggle full screen Open in new window Toggle example guides Toggle HTML markup

Block for creating layouts. Suitable for columns, cards and horizontal lists. By default, inline-block technology is used for layouts. Using D utility, you can change the layout technology to flexbox.

Parts of the layout within a row are created using the elements:

  • row__col
  • row__item

Rows can be put into each other an unlimited number of times.

Row contains CSS counter named ml.row-item.

3.4.1 #core-blocks.row.row__col row__col

Toggle full screen Open in new window Toggle example guides Toggle HTML markup

The main element for creating columns in layout. By default, it stretches the entire width of the container. The columns have gutters on the sides and bottom.

The gutters are implemented using the margin and CSS custom properties. If var() is not supported by the browser, then fallback are provided:

margin-right: 16px;
margin-left: 16px;
margin-right: var(--ml.gg);
margin-left: var(--ml.gg);
margin-bottom: 2rem;
margin-bottom: calc(var(--ml.gg-rem) * 2);

The size of gutter fallback you can find in the settings in ml.$grid-default-gtr.

Column width is set using W utility. See example above. The formulas for calculating the width and offset for the column are stored in the custom properties. In --gc-rule for W and --mgc-rule for Ml utility. These formulas:

--gc-rule: calc((var(--gc) - 0.0001) / var(--ml.grid-cols) * 100% - var(--gc-diff) * 2);
--mgc-rule: calc((var(--mgc) - 0.0001) / var(--ml.grid-cols) * 100% + var(--gc-diff));

--gc or --mgc is the value of the width or offset in the grid columns. This is a custom property of the utility.
(var(--gc) - 0.0001) - fix for calc() bug in Edge 16-18.
--ml.grid-cols - the number of columns in the grid system. This is a global custom property that can be changed on any DOM node.
--gc-diff - variable is equal to the gutter if the gutter at the columns is created using margin. In row__col is equal 1gg.
Some of these variables can be changed using custom utilities.

You can create nested layouts with a mix of row and row__col on the same DOM node. If you don't need double fields in the nested row, use the mix with the row__item. See example below.

All examples of the work of nested layouts are here.

3.5 #core-blocks.wrapper Wrapper

Toggle full screen Open in new window Toggle example guides Toggle HTML markup

The block-container for the main content of the page. The maximum width is set in the settings. The default is 1200px.

The container has sides padding that equal to gutters so that the content does not stick to sides on small screens. On wide screens the container is centered.

Like in row__col, in wrapper is used var() fallback for fields:

padding-right: 16px;
padding-left: 16px;
padding-right: var(--mlu.gg);
padding-left: var(--mlu.gg);
