7.1.13 #sass-tools.functions.map_getters Getters from utils map

Functions for conveniently retrieving values from a map of utility values. They help to use utilities values in common CSS rules. If there is no value in the map, an error will occur.

Functions names have the same form: mlu-#{utility name}. Each function has 1 argument - a key in the map. Each function corresponds to a map whose name has the form of a mlu-#{Utility name}-map. By default, these maps are empty. To start using the functions, add your values to the map you need, as in the example below.

$C-map: (
	"f": #fff,
	"mn1": #c05,
);

$mlu-C-map: $C-map;

mlu-c("mn1") => #co5

Available functions and maps corresponding to them:

  • mlu-c - $mlu-C-map
  • mlu-fz - $mlu-Fz-map
  • mlu-bgi - $mlu-Bgi-map
Parameters:
  • $key
    key for the value in the map.

Bpm is breakpoint manager.

	.inp--common {
		padding-left: 48px;
	}

@include mlu-bpm($to: "lg") {
	.btn--common {
		margin-left: 48px;
	}

@include mlu-bpm("sm", "lg") {
	.btn--common {
		margin-top: 48px;
	}
}

Compile into:

@media (min-width: 760px) {
	.inp--common {
		padding-left: 48px;
	}

@media (max-width: 959px) {
	.btn--common {
		margin-left: 48px;
	}
}

@media (min-width: 520px) and (max-width: 959px) {
	.btn--common {
		margin-top: 48px;
	}
}
Parameters:
  • $from

    Defaults to: false
  • $to
    breakpoint. Can be keyword (string) or number. Generate to 'max-width' - 1 mediaquery. Together with $from generate to 'min-width and max-width - 1'.

7.2.6 #sass-tools.mixins.mk_util mlu-mk-util

Сreates utilities based on a map in which the keys are the values of the utilities, and the values are the values of the CSS properties.

Utility values are set according to the utility naming rules. They can be a string (including empty) or a number. Also, the value can indicate the breakpoint on which it should work - "tb_lg". Even if the breakpoint is passed as a parameter when this mixin is called, then the data about breakpoints from the values map will be more priority. In addition to a specific breakpoint, 2 special values are valid:

  • bp - create a value only on breakpoints: "r_bp", "4gg_bp"
  • all - create a normal value and on breakpoints: "ib_all", "_all".

Values for CSS properties can be any CSS values, as well as numbers with gg and su units. Values with gg units are created using the mk-gtr mixin.

For most commonly used utilities, there are mixins-wrappers in which utilities names and CSS properties are already selected.

$Bgc-map: (
	"f": #fff,
	"mn0_lg": #60f,
	"mn1": #c05,
);

@include mlu-mk-util("Bgc", "background-color", $Bgc-map);

CSS

.Bgc-f {
	background-color: #fff;
}

.Bgc-mn1 {
	background-color: #c05;
}

@media (min-width: 960px) {
	.Bgc-mn0_lg {
		background-color: #60f;
	}
}
Parameters:
  • $name
    the name of the utility. It can be specified as a CSS class (with a dot) or just a string.
  • $prop
    CSS property name.
  • $map
    map with utilities names values and CSS properties values.
  • $bp
    breakpoint on which utility values will be generated. Can be any breakpoint from the settings or a special value: bp and all.
    Defaults to: false
  • $states
    list of state names. For each value of the utility, all states from this list will be generated. Info about states see here.
    Defaults to: false
  • $direct
    service argument which used only by mk-util-directions.
    Defaults to: null

7.2.7 #sass-tools.mixins.mk_util_dir mlu-mk-util-directions

Creates utility groups that have different directions. For example margin and padding.

The mlu-mk-util functionality is mainly used, but the map of values for creating utilities is slightly different. Before the utility value, one or two letters indicate the direction. Keywords and variables are separated by a hyphen from direction. Examples: l4su, xy1gg, t-b8_lg.

A mixin can create positioning utilities that match the CSS properties: top , right etc. To do this, just pass the map with the data. Each of the 4 main directions will correspond to the CSS property. If the direction is combined, then a modified utility mX, mY or mXy will be created. These utilities changes several properties. For the convenience of creating positioning utilities, a mixin-wrapper is used.

All parameters are the same as in mk-util. Only their order and default values differ.

$Xy-map: (
	x3gg: 3gg,
	l4su: 4su,
);

@include mlu-mk-util-directions($Xy-map, "P", "padding");

@include mlu-mk-Xy($Xy-map);

CSS

.Px3gg {
	padding-right: 48px;
	padding-right: calc(var(--mlu-gg) * 3);
	padding-left: 48px;
	padding-left: calc(var(--mlu-gg) * 3);
}

.Pl4su {
	padding-left: 16px;
}

.mX3gg {
	right: 48px;
	right: calc(var(--mlu-gg) * 3);
	left: 48px;
	left: calc(var(--mlu-gg) * 3);
}

.L4su {
	left: 16px;
}
Parameters:
  • $map
    map with utilities names values and CSS properties values.
  • $name
    the name of the utility.
    Defaults to: null
  • $prop
    CSS property name.
    Defaults to: null
  • $bp
    breakpoint on which utility values will be generated.
    Defaults to: null
  • $states
    list of state names.
    Defaults to: null

7.2.12 #sass-tools.mixins.util_wrappers mk-util wrappers

Mixins-wrappers over mlu-mk-util and mk-util-directions, in which the names of utilities and CSS properties are written. Only a map with values is required. All have the same parameters.

The names of all mixins begin with mlu-mk- and end with the name of the utility they create: mlu-mk-Maw, mlu-mk-Fw etc. The exception is mlu-mk-Xy, which creates positioning utilities. 2 hyphens are written in the names of mixins for custom utilities: mlu-mk--Gcc. Mixins for groups of utilities with directions are marked separately.

Mixins for common utilities:

  • Positioning: Z
  • Box-model: W, Maw, Mih
  • Typografy: Fw, Fz, Ff, C, Lh
  • Visual: Bgc, Bdc, Bdw, Bdrs, Trf, Op

For custom utilities:

  • -Gcc

For utils with directions:

  • Xy
  • M
  • P
  • Bdxy

Example:

$Bdc-map: (
	"sc2": #333,
	"gr0": #eee,
);

@include mlu-mk-Bdc($Bdc-map);

CSS

.Bdc-sc2 {
	border-color: #333;
}

.Bdc-gr0 {
	border-color: #eee;
}
Parameters:
  • $map
    map with utilities names values and CSS properties values.
  • $bp
    breakpoint on which utility values will be generated.
    Defaults to: null
  • $states
    list of state names.
    Defaults to: null
