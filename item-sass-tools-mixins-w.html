7.2.13 #sass-tools.mixins.w mlu-W

Set width on different breakpoints. It works similarly like W utility.

.block {
	@include mlu-W(2c, "sm");
	@include mlu-W(3gc, "md");
	@include mlu-W(4gs, "lg");
}

.foo {
	@include mlu-W(2gc, "sm", $css-vars: false);
}

CSS

@media (min-width: 520px) {
	.block {
		--gc: 2;
		width: calc(16.666% - 32px);
		width: var(--gc-rule, 16.666%);
	}

	.foo {
		width: calc(16.666% - 24px);
	}
}

@media (min-width: 760px) {
	.block {
		width: calc(25% - 32px);
		width: calc(25% - var(--mlu-gg) * 2);
	}
}

@media (min-width: 960px) {
	.block {
		width: 33.333%;
	}

	.foo {
		width: calc(16.666% - 32px);
	}
}
Parameters:
  • $width
    can be in variable columns(c), grid-columns(gc), grid-spans(gs), any CSS units(px, rem etc), auto or inherit. gc and gs calculated to % according to the formula 'n / $cols * 100%' and rounded by mlu-decimal-floor.
    If $width in gc and $css-vars = false, width will generated with considering different gutters on breakpoints. As in the example above foo.
    If $width in c and $css-vars != 2, will be generated fallback in gc.
  • $bp
    breakpoint on which $width will sets. Can be number or breakpoint name.
    Defaults to: $mlu-bp-xs
  • $cols
    columns count in grid. Can be number or breakpoint name.
    Defaults to: $mlu-grid-default-cols
  • $gutters
    map with gutters values on breakpoints. Map item example: "sm": 12px. Custom gutters only work when the $css-vars = false.
    Defaults to: $mlu-grid-gutters
  • $calc
    bool flag for generation CSS calc() fallback. It can needed only if $width set in 'gc' unit.
    Defaults to: false
  • $type
    service argument which used only by mixins-wrappers. It needed for reusable mlu-W for generating grid offset, because the algorithm for generating them is the same.
    Defaults to: "width"
  • $css-vars
    use CSS custom properties for gutters. Works as a similar flag in the settings.
    Defaults to: $MLU-FLAG-CSS-VARS
