5.1 #core-styles.css_vars CSS variables

Global CSS custom properties which used in MLUT. Their names are stored in variables and can be changed in settings. Names of such variables correspond to the original names of custom properties. Global custom properties names begin with the mlu prefix.

The values and meaning of many custom properties correspond to the SASS settings of MLUT core. But unlike them, custom properties can be changed dynamically without recompiling.

Custom properties are used as a progressive enhancement. By default, fallback is enabled for browsers that do not support them. Generation of custom properties is controlled by $MLU-FLAG-CSS-VARS.

Remember that custom properties do not work in all browsers. This means that part of the settings, such as dynamically changing the gutters and the number of columns in the grid system, also does not work everywhere.

In some blocks and utilities, custom properties are also used. Look for info about them in the description of the corresponding block or utility.

You can find global CSS variables in sass/core/variables.scss.

Parameters:
  • --mlu-gg
    grid gutter in px. Value sets here. Fallback is $mlu-grid-default-gtr.
    Defaults to: 12px
  • --mlu-gg-rem
    grid gutter in rem.
    Defaults to: 0.75rem
  • --mlu-grid-cols
    number of columns in the grid system. Sets by $mlu-grid-cols-xs and changes on breakpoints according to the settings.
    Defaults to: 4

5.3 #core-styles.system_font System font

This is a directive @font-face that creates a font called system-font. System-font set native font family on user's OS. Use it for safety in font-family with custom web font:

font-family: "PT Sans", system-font, Arial, sans-serif;

It turned off in default MLUT distributive.
